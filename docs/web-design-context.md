# PerformanceProtocol Web Design Brief

This document is a self-contained product and data brief for designing the PerformanceProtocol web app. It describes what the interface should support and what data is available. It is not an implementation guide.

## Product

PerformanceProtocol is an endurance training platform for serious athletes. It closes the loop between device data, a selected training methodology, an executable training plan, day-to-day completion, and adaptive weekly adjustments.

Current scope:

- Road running, focused on marathon and half-marathon training.
- COROS-first import and calendar sync.
- Phone OTP authentication.
- Multi-user-ready API and web app.

Planned scope:

- Trail running.
- Triathlon.
- Long-distance cycling.
- User-created or extracted training skills.

Do not use `ST` as a product name. `ST` is only a legacy internal codename for package paths, environment variables, and other compatibility identifiers.

## Differentiators

1. **Bring-your-own methodology.** A skill is a pluggable training methodology. The same platform can generate plans from different skills.
2. **Closed-loop execution.** The platform pulls device history, pushes confirmed workouts to the athlete's calendar, and compares planned work against completed work.
3. **Adaptive adjustments.** Weekly recommendations can modify future workouts while preserving completed history.

## Tech Reality

- Backend: Python 3.11, FastAPI, SQLAlchemy, SQLite locally, Postgres in production.
- Frontend: Next.js 14 App Router, TypeScript, Tailwind CSS, SWR.
- API: REST JSON.
- Auth: phone OTP and bearer token.
- Production: Fly.io, with `performanceprotocol.io` and `api.performanceprotocol.io`.
- Mobile-first, with desktop layouts for planning and review.

## Skill System

| Skill slug | Display name | Origin | Tags |
|---|---|---|---|
| `marathon_st_default` | PerformanceProtocol Marathon Plan | Built-in fallback | hybrid, base-build-peak |
| `coach_zhao_unified` | Coach Zhao Unified Marathon Methodology | Distilled from athlete COROS history | seasonal, northern-china, low-LSD, working-professional |
| `running_beginner` | Beginner Runner Plan | Built-in fallback for low-history athletes | beginner, run-walk, rpe |

Each skill has:

- `slug`
- `name`
- `version`
- `sport`
- `supported_goals`
- `description`
- `author`
- `tags[]`
- long-form `skill.md` methodology content

The web UI should:

- List skills with tags and one-line descriptions.
- Show the full methodology document.
- Let the athlete choose which skill generates the next plan.
- Show which skill produced the active plan.

When switching skill mid-cycle:

1. Completed and missed workouts stay frozen.
2. Today-forward workouts are regenerated by the new skill.
3. The new skill must be applicable to the athlete, goal, and remaining weeks.
4. The UI must show counts for frozen and regenerated sessions before applying the switch.

## Core Data

### Athlete

- `id`
- `name`
- `sport`
- `level`
- `weekly_training_days`
- static profile details when available

### Race Goal

- sport and distance
- optional target time
- race date
- training start date
- plan weeks
- feasibility status

### Training Plan

- title
- mode
- week count
- start date
- race date
- target time
- status
- confirmed flag
- `skill_slug`

### Workout

- scheduled date
- week index
- discipline
- workout type
- title
- purpose
- duration
- distance
- pace, HR, or RPE targets
- adaptation notes
- status
- interval steps

### Activity

- start time
- discipline
- distance
- duration
- pace
- HR
- training load
- matched planned workout when available

### Adjustment

- reason
- recommendation
- affected workouts
- status
- effective date range

## Primary Workflows

Daily:

- See today's workout quickly.
- Open full workout detail with steps, targets, and notes.
- Import or refresh COROS activity data.
- Compare completed activity against the planned workout.
- Mark complete, partial, or skipped.

Weekly:

- Review planned vs executed volume.
- See upcoming structure.
- Apply or reject proposed adjustments.

Occasional:

- Connect COROS.
- Create a goal.
- Generate and confirm a training plan.
- Sync workouts to COROS.
- Switch skill.
- Read methodology documents.

## Design Direction

- Serious, precise, data-rich, and restrained.
- Avoid generic AI-startup visuals and decorative gradients.
- Use real training signals: weekly volume, workout status, readiness, device sync, and plan adjustments.
- Keep daily mobile workflows fast.
- Use `PerformanceProtocol` consistently wherever the product name appears.
